## プロダクトマネージャーがCursorと作る、"思考が蓄積する"仕事環境 ~AI伴走を当たり前に~

[Naomi Shiraishi](https://note.com/naomix)

プロダクトマネージャー業務は、日々のタスクから中長期の戦略策定まで、多岐にわたります。忙しさに追われ、その日の思考や課題が記憶の彼方に消えてしまう…  
私はそのスパイラルに入ってしまっており、タスク管理ツールやtimesに「日報」を書いたりと試行錯誤していたものの、何度も挫折していました。

## Cursorとの出会いと活用のひらめき

そんな中、社内で利用が広まっていたAIエディタ「Cursor」。  
当初は、ノンエンジニアでもコード調査やSQL作成が出来ることや、一貫したドキュメント作成が出来ることに魅力を感じて使っていました。

これまで私のAI活用は、「PRDの壁打ち」に代表されるドキュメント作成の補助、「開発仕様の確認」など特定の複雑な業務だけにとどまっていました。  
小さな日常的タスクにもAIを絡められる可能性があるとは、正直あまり想像できていなかったのです。

きっかけは、みやっちさんの [Cursorエージェント講座 超応用編 - AIエージェントに「心」を理解させられるか？ -](https://www.docswell.com/s/miyatti/KN182G-2025-03-21-202746) という登壇資料。  
これにより、「特定業務だけにAIを活用する」という従来の思考から、「自分の業務にフィットするようにAIを設計する」事自体を、AIと一緒に進められるのではないか？」と考えるようになりました。

## 基本的なセットアップ

「業務に伴走させるならTodoを起点にするのが良さそう」というアイディアと、「前々から課題感のあった振り返りの仕組み化も絡めたい」というゴールを立てて、取り組みを開始しました。  
  
まずCursorでタスク管理と振り返りを行うためのフォルダを作成し、YYYYMM\_FileName形式の.mdファイルをそれぞれ作成。  
ちなみに最近だとNotionも公式MCP（外部ツールへ連携できる仕組み）も公開されましたし、ローカルに拘る必要はなくスタイルにフィットする方式で良いと思います。

```python
📁 Cursor/
├── 📁 todo/ 　　　　　# タスク管理用フォルダ
├── 📁 retrospective/ 　# 振り返り用フォルダ
└── 📁 log/ 　　　　　  # 行動ログ用フォルダ
```

ToDoには、「ルーティンワーク」「長期タスク」「短期タスク」の見出しを設け、短期タスク配下には日付の見出しを立ててタスクを書き出します。

### 伴走方法を言語化

Cursorは比較的ざっくりとした依頼でもそれなりに作業を進めてくれますが、イメージする業務ステップが明確な場合は、Rules（Cursorに一貫してほしい振る舞いや前提を定義する機能）に定義しています。

みやっちさんの発信を参考に、目的とそれぞれのタスクのステップを記載し「1.が終わったら2.を開始するか提案する」、のように **流れが分断されない形を意識しました** 。  
画像はシンプルバージョンですが、ファイルの命名規則や見出しの追加場所など好みに合わない挙動をした場合はこちらに指定を追加していきます。  
\`Cursor > General > Project Rules\` から追加できます。

```python
#目的
- 日々のタスク管理の効率的な進行
- 適切な振り返りの実施
- 行動や思考ログの記録

## 具体的な流れ
1． 「今日のタスク」と依頼されたら、todoリストをチェックして、繰り返し・長期的・短期的タスクを確認し、手助けをして。
2. 「振り返りをして」と依頼されたら、todoリストの内容と、チャット上の記録を元にその日の振り返りを促して。効果的な振り返りをするための洞察に溢れた質問をし、その回答を踏まえて振り返り内容の提案をして。内容が承認されたら、<振り返り用ファイル＞に追記して。振り返りが終了したら、行動・思考ログの記録を提案して。
3. 2.でYESと回答される、または「行動を記録して」と依頼されたら、todoとretrospectiveからその日の行動や思考に関わる必要な情報のみを抽出して<行動ログファイル>に記録して。記録が終了したら次の営業日用のtodo作成を提案して。
4.3.でYESと回答される、または「明日のタスクリスト作って」と依頼されたら、次の営業日用の短期タスクのリストを作成して。短期タスク内に最新日付が先頭となるように、その前のタスクリストでチェックが付いていないものをコピーしてして、最後にその他追加するべきものがないか質問して。
5.一連の振り返り・行動記録タスクの終了が宣言されたら、親密さを含んだ文体で、良い一日の終りが迎えられるように今日の頑張りへの称賛や、明日へ前向きなメッセージを話して。
6. 「新しい月リストを作って」と指定されたら、Log、retrospective、todoファイルを、どの月のファイルを作成するか問し、回答があった月をYYYMM形式で表現し、YYYYMM_filename.mdの形式でコピー元と同じ場所に新しく作成して。
```

## タスク管理ワークフローの実践

todoファイルを@指定して「今日のタスクを進める手助けをして」と指示、タスクの優先度や手助け提案を受けます。  
質問に答えたり、追加情報を与えたりしながらタスクを進めていきます。  
  
その後は「<タスク名>終わり」と伝えてチェックボックスをDoneにしたり、このタスクはここまで進んだと伝えて更に細かいレベルのタスクに分解するのも一瞬です。

1日の作業が終わったら、「振り返りをして」と依頼します。  
こなしたタスクをリストアップしながら、振り返りの手助けをしてくれるので、質問に答えたり、タスク中での気づきや課題、エピソードを書いていきます。内容がまとまったら、振り返り用のファイルに日付の見出しを立てて記録します。

## 少し進んだ活用例

基本的な進め方に慣れてきたら、こんな活用もおすすめです。

### カレンダーの予定を渡して精度を上げる

Googleカレンダーの予定リストをCursor渡すことで、MTG Dayなのか集中作業できる日なのかを踏まえた提案が可能です。  
行動ログにも会議予定が反映されるので、よりログの精度が上がります。  
現在はGoogleカレンダーからSlackに連携される予定リストをコピペする手運用のため、ここも自動化できればさらにライトに活用できそうです。

### 明日の仕事の発射台を作る

明日のタスク洗い出しが終わったら、「このタスクに取り組むための具体的ステップを整理しておいて」や「参考になる情報をMCPで繋げた社内情報から集めておいて」と頼んでおくと、次の日の仕事の初速が格段に上がります。  
Cursorだけでは完結しないことももちろんありますが、朝から集中して取り組めるようになりました。

### 評価や中長期の振り返りが自由自在

行動ログや振り返りのファイルをまとめてCursorに指定し、「4月の学びや重要な活動をまとめて」と依頼すると、月次の活動サマリーが作成できます。同様に、半期や通期など、指定するファイルを複数月にすればより長い時間軸での振り返りも可能です。  
特に人事評価シーズンの自己評価作成時には、実績確認や成長ポイントの抽出が容易になり、大きな助けとなることが期待できます。

## Cursor導入で得られた変化と効果

Cursorを使い始めてまだ日は浅いですが、すでにいくつかの明確な変化を感じています。

### 1\. AIが日常業務に自然に溶け込んだこと

これまでAIはPRDの壁打ちなど、特定の「大きな」タスクで使うもの、という意識がありました。しかしCursorでは、タスク管理のプロセス自体にAIが組み込まれているため、会議の準備、インタビュー設計のアイデア出し、リリースノートの下書き、ちょっとした開発影響範囲の調査など、 **大小問わず日々のあらゆる活動に、自然にAIを「相棒」として活用するように** なりました。

### 2.「経験」を言語化し、「歴史」として捉える習慣がついたこと

もともと、効果的な振り返りのためにCursorでの記録を始めましたが、日々AIと対話しながら活動を記録し、思考を言語化するプロセスは、想像以上にスムーズです。  
最近読んだ企画の本に「『愚者は経験に学び、賢者は歴史に学ぶ』と言う格言があるが、自身の企画から、共通点や方程式を見出し、経験を歴史化できているか」という箇所があり、 **日々の行動ログは、単なる経験の記録ではなく、未来の糧となる「自分だけの歴史」を紡ぐ行為** なのだと気づきました。このフレーズにより、自分の試行錯誤が肯定されたように感じられ、前向きにこの取り組みに向き合えています。
