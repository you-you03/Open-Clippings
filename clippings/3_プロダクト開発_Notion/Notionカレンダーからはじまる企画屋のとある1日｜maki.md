[maki](https://note.com/maki_23)

こちらの記事は「Notion Community Advent Calendar 2025」の23日目の記事です。

昨日は [nobunosuke](https://note.com/nobunosuke/n/n44f9c22ad723) さんの [「](https://note.com/nobunosuke/n/n44f9c22ad723) [Notion](https://note.com/nobunosuke/n/n44f9c22ad723) [オタクへ。ショートカット対決、しませんか](https://note.com/nobunosuke/n/n44f9c22ad723)[?](https://note.com/nobunosuke/n/n44f9c22ad723)[」](https://note.com/nobunosuke/n/n44f9c22ad723) でした。  
めちゃくちゃ勉強になりました…！自分、初級でした！中級のひとつめしか使いこなせていませんでした笑

ショートカット技は初級な私ですが、Notionユーザーになってもうじき8年目突入です。もはやNotionのない人生なんて考えられない！  
Notionアドベントカレンダーへの参加も、今回が3回目になりました。 [田原さん](https://x.com/shogocat/status/1987106937067217355?s=46&t=AuhAXSO9f_FyorzRRy--RA) 、いつも素敵な企画を、本当にありがとうございます！

なので今年は、 **仕事で（PC＆iPadで）がっつり使っている様子** をユースケースとして纏めてみようと思います。よろしくお願いします！

  

## 1.実録：企画屋のとある1日

本気の実録をやろう！と決めてまめにログってみたらボリュームが多くなりました。すみません。  
2025年12月のとある月曜日。 **MTGが少なめで企画作業が多め** の日を狙って、Notion活用の実態をログってみました。

［私の仕事：ざっくり紹介］

> ・事業開発寄りの企画職  
> ・事実上の上司はCSO。メンバーはいない1人チーム状態  
> ・四方八方から0→1的な案件が降ってくる、社内フリーランス的なポジション  
> ・Notionをワークスペースにしており、1人で活用している

仕事で重宝しているのは、何と言ってもNotionカレンダー！24年8月に書いたnote「 [Notion](https://note.com/maki_23/n/n7369da5d7157) [カレンダー導入後](https://note.com/maki_23/n/n7369da5d7157) [7](https://note.com/maki_23/n/n7369da5d7157) [ヶ月の活用状況を語る](https://note.com/maki_23/n/n7369da5d7157) 」から、使い方はあんまり変わってないのですが。  
Notionカレンダーがリリースされた2024年1月以降、2年弱活用していますが、すっかり「なくてはならない存在」です。

こちら▼がログった週のNotionカレンダーです。

![画像](https://assets.st-note.com/production/uploads/images/236585028/picture_pc_dd1144953fd6f8014065ee7cfccd5dda.png?width=1200)

ピンク色＝〆切（納期）。案件やMTGはアイコンで表現

月曜朝にスクショを撮ったので、まだ“綺麗”な画面。笑 最後に、この1週間が終わった後の同画面のスクショを載せていますが、タスクとタスクが重なってたり（内職＆予定通りに終わらず）、ショートMTGが追加されまくっていて、やっぱり予定通りには進まないなあ、と実感します。

私の作業スペースはほぼ9割がNotion。Notionカレンダーを開いて、予定の「Notionで管理」を押して、すべての作業やMTGが始まります。

## 2.12月のとある月曜日

ありのままの実録を、特にNotion上で作業した時間を中心に綴ってみようと思います。  

### 8:30～9:10／朝礼×Notionカレンダー

```
スケジュール管理：当日の予定を確認
```

・駐在オフィス朝礼での、各自の「本日の予定」発表時、Notionカレンダーを見ながらMTG予定＋企画作業タスクを発表  
・週末に関係者から降ってきた細かな依頼事項や自分の思いつきは、 **週末の自分がNotionカレンダーにタスク登録してくれている** ので、内容を微調整  
・3本の朝礼をハシゴしつつ、iPadで「今日の手書きノート」を用意。NotionのDASHBOARDに表示させている「今日のタスク」スクショを撮り、GoodNotesの「今日のページ」に貼る。 **1日1ページを使用。閃きや打合せメモをがしがし手書きで書いていくスタイル**

![画像](https://assets.st-note.com/production/uploads/images/236585396/picture_pc_02f922dc04bc612063cfecadf235c36a.png?width=1200)

1日1ページ使用

> 月曜朝は特に、Notionカレンダーを見ないと（今日の予定なんだっけ？？？）状態です。土日は仕事モードを完全シャットダウンしているので、「記憶、思考がリセット」されてしまう感覚です。でもNotionカレンダーがトリガーになってくれて、先週金曜の自分と、月曜朝の自分の脳が繋がります。

  

### 9:10／今週の〆切確認×Notionカレンダー

```
スケジュール管理：今週のMTG＆作業予定を、Notionカレンダーで確認
```

・納期に対してタスク設定漏れがないか確認  
・実現不可能では？？？と過密スケジュールになっている部分は、優先順位が低いタスクを翌週に移動

> 「時間」というブロックを綺麗に整列させるイメージで、スケジュールを整えます。不自然な重なりが続いていたり、タスクの順番がおかしいと途端に作業が滞るため。この整列作業中に、案件をスムーズに進めるための関係者への連絡・調整を思いついて、タスク追加したりもします。

  

### 9:15／動画編集の指示出し×Notion

```
コンテンツ系：インタビュー動画 使用場面のピックアップ作業
```

・30分のインタビュー動画を、3〜5分尺に収める為に、使用場面をピックアップする作業 ×3本  
・インタビューの書き起こし原稿を、Notion（DB）にシーン区切りでぶち込んであるので  
①使用したい場面をDB上で赤文字にし  
②選択プロパティで「採用」と書き、フィルタリング  
③採用パートの秒数を数字プロパティに記入し、秒数を合計  
・採用シーンの動画を見ながら、尺に収まるよう調整し  
・制作会社に使用シーンを指示

> この段取りで、想定よりかなり速く作業を進めることができた感があります。動画系の作業はとにかく時間がかかる印象がありましたが、［2時間で終わらせる為の段取り］を考えて着手したのが良かったなと思います。  
> Notion上で作業しているので、後日の微調整時もすぐにDBを参照できて良きでした。

**［11:29／休憩］**  

### 11:34／MTG準備×Notion

```
MTG系：MTG用の資料準備（事前に展開）

・夕方にある「資料修正箇所の確認MTG」の内容確認  
・参考にする過去の資料をピックアップ  
・事前チェックよろしく！と参加メンバーに連絡  
・連絡した内容を、Notionカレンダー→MTG予定ページを開き、議事録DBに転記しておく

> 議事録DBは、テンプレートを何種類か登録してあります。参加者やMTGのタイプ別に、テンプレートを作成しています。（今は12種類ほど）

### 11:46／社内チャット確認×Notion

```
作業準備系：チャット内容を確認＆返信
```

・返信内容＆チャットのリンクURLを、当該案件のNotionページに転記  
・当該案件の企画仕事に着手した時、改めてチャットを探さなくていいようにする作業

> 地味にこの作業が大事だなと思っていて、よし作業するぞ！という時に「あの情報どこにあったかな？」と出鼻を挫かれる時間＆ストレスが無いのが良きです。

```

・取り直した新幹線の予約内容をNotionカレンダーに転記

> こういう情報も、Notionがなかったらメールをいちいち探したり、端末ごとに異なるカレンダーを確認する手間があるのかと思うと…Notionに感謝。

**［12:14／昼休憩］**

### 12:39／試験勉強×Notion

```
情報収集系：来年2月に受験予定の試験勉強
```

・Notionページに、過去問を解いた際に「わからなかった／間違えた単語」リストを作成  
・単語を検索して解説テキスト＆図解探してコピペ

> 溜め込んだ情報は、後日、手帳に写し直す予定。こうして作業を分解すると、苦手な勉強も、無理なく効率的に進められるものだなあと学びました。

  

### 13:01／資料修正×Notion

```
戦略系：中期経営計画の資料作成（会議での指摘箇所の修正作業）
```

①先週末の会議で話し合われた内容を、当日中に参加者からFB聞き取りしてメモしてある。その手書きメモがNotionの作業ページに貼られていたので（貼ったのは過去の自分）、まずメモを眺める

> 先週の自分がメモを貼ってくれてなかったら、まずメモをどこに書いたか？から探さなければいけなかったので、過去の自分ありがとう…！と毎回拝んでいます。

②上司にアポ取りTEL、修正→確認→修正→提出の段取りをつける。アポ取得→Notionカレンダーに予定登録。既に入れてた作業タスクを翌日に移動  
③MTG議事録ページ作成（テンプレート反映）。 **これから行う作業のページリンク** を貼っておく（MTG時に作業データを画面共有できるように）  
④資料修正作業。PPT修正→ **修正メモをMiroに記載し、MiroリンクをNotion作業ページへ埋め込み**  
⑤関係者が作成したデータを分析する必要が出てきたので、ExcelをCSV化してNotionにインポート  
⑥DBにして、viewを複数作成し、いろんな角度でデータをフィルタリングしながらPPTに数字を記入  
⑦社内チャットにて、PPTを上司へ提出。 **提出時の申し送り事項＆④のMiroリンクを、③の議事録ページに貼っておく**

> PPTやExcelを使わざるを得ない作業は極力短く！が信条です。NotionもしくはMiro、iPad（GoodNotes）で「考え仕事」をするようにしています。それ位PPT／Excel作業が嫌いです。「まあでも、昨日の自分が作業指示を残してくれてるし、無心でPPTで清書すればいいか…」という気持ちで作業開始すると早く終わります。 **昨日の自分が今日の私の上司** です。  
>   
> また、「どんな修正を加えたのか？どんな疑問出しをしたのか？」すら2日後には忘れてしまうので、作業メモ（ダイジェスト）や申し送り事項も、次回タスクのページに記すようにしています。

＊私の場合、こういう **「消えモノ系（瞬間的に必要になるデータ）作業」が多いので、Notionカレンダー軸の シンプルなDB構造がベスト** なのでした。

```

> 過去の見積書や、案件終了後に精算した際のデータも **Notion上に大体残っているので、データを探す時間がかからない** のが最高です。

### 15:59／定例MTG×NotionAI

```
MTG系：壁打ちをしてくださる顧問との週一1on1

## 3.さいごに

Notion×Notionカレンダーの活用で、こちらのpostの状態を実現できているなと感じます。  
「ひとり事業開発担当」を続けていられるのも、「情報の置き場所」をNotionに全部委ねていて、引き出しをNotionカレンダーにしているから。

改めて「企画仕事の1日」をログってみて思うことは、企画仕事で“無駄”なことが3点あって、これを解消できるのがNotionだなと実感しています。

> ［企画仕事にいらない要素］  
> ・作業に着手する時、関連する情報を探す時間  
> ・見つからなくて、細かな閃きを逃してしまうストレス  
> ・閃きをストックできていないが故に「思い出さねば！」と唸る時間

あと、私は整理整頓が上手い方ではなく、最近は特に、Notion上の情報の構造化もできていないな…と感じています。  
なので **Notionカレンダーの「タスク DB」に一元管理できている現状** が、自分にとってのベストなのかなと。先々のタスクやMTGのページにリンクを貼りまくって先回りすることで、未来の自分が作業着手する際に困らないようにしている、その繰り返し。  
**Notionカレンダーを軸にしたシンプルな使い方** だからこそ、抱える案件が増えても、混乱せずに回せているのかなと思います。  

Notionユーザーになって7年経っても、まだまだ“進化”できるんだなと思わせてくれる最高の相棒、Notion。  
来年はNotion AIエージェント活用にトライしてみたいなあと思わせてくれたこちらのnote、最高に面白かったです！

今は **「過去の自分×Notion」が現在の自分の秘書役** だけど、いつかNotionが単体で秘書になってくれると、さらに自分のキャパが拡がるかも？と思えました。運用を考えてみようかな。ワクワクします！
