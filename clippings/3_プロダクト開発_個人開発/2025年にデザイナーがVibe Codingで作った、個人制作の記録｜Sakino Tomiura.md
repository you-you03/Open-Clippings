## 2025年にデザイナーがVibe Codingで作った、個人制作の記録

[Sakino Tomiura](https://note.com/sakinotomiura)

2025年は、Vibe Codingがさらに広がり、コードを書くことが、より多くの人にとって身近になってきた年でした。  
そんな流れの中で、「コードをほぼ書けないデザイナー」である自分自身は、思いついたものをとにかく作る、ということをひたすら繰り返していました。  
  
Vibe Codingで、自分用のツールをいくつも作りながら、試したことや考えたことを積み重ねてきた一年です。

このnoteは、その中で何を作ったのか、どんな試行錯誤をしていたのかをまとめた記録です。

  

## 「思いついたら作れる」ようになった2025年

### まずは頭に浮かんだものを、小さくたくさん作った

Vibe Codingで「なんとなくできること」が分かってきて、 **自分のデザインをベースに、作りたいものを実際に作れる** ようになったのが2025年の春ごろでした。

そこから、V0、Cursor、Lovable、Bolt、Replit…など、さまざまなツールを試しながら、ひたすら手を動かしてきました。このタイミングで、Vibe Codingに関するという [簡単なLT](https://x.com/sakinotomiura/status/1930882122933588347?s=20) もやらせていただきました。

![画像](https://assets.st-note.com/production/uploads/images/238211045/picture_pc_e7b4eb2e48bc3c00d7ab53bd0ccc09ec.gif?width=1200)

![画像](https://assets.st-note.com/production/uploads/images/238206238/picture_pc_d49f3c631adab1156ecff8b7484e7cbe.gif?width=1200)

### 次は、自分が使うものを作ろう

遊びとしてのツールを一通り作ったあと、次は「自分が本当に欲しいもの」「自分のテンションが上がるもの」を作ろう、という目標を立てました。

ただ作るだけではもったいないので、 **学びたいこと、研究したいこと、挑戦したいことをそれぞれのプロジェクトごとに設定** しながら、個人開発を進めてきた期間でもあります。

How-toや再現手順はありませんが、デザイナーがVibe Codingを使って、実際にどんなものを作ってきたのか。そのアウトプットを中心にいくつかまとめました。

  

## この半年のアウトプット

### 🍳 レシピストックアプリ

![画像](https://assets.st-note.com/production/uploads/images/238211663/picture_pc_df9cbb78627410ed6d6e6940d103a80a.gif?width=1200)

料理が趣味なのですが、 **レシピって本当に散らかる** んです。レシピサイト、レシピ本、Instagram、YouTube、自分のメモアプリ。媒体も形式もバラバラで、すぐに迷子になります。

それをどうにかしたくて、 **URLや写真からレシピを抽出し、同じフォーマットで保存・閲覧できるアプリ** を作りました。保存時には自動でタグ付けもされます。

基本的な機能を実装したあとも、自分が欲しい機能を追加しながら、今も使い続けています。

> 💻️ 使ったツール: Cursor / Figma / Nano Banana Pro  
> 📚️ テーマ: AIを介したレシピ抽出ロジック設計と、DB・認証連携の実装

### 🎨 ポートフォリオ＋CMS

![画像](https://assets.st-note.com/img/1766240257-lnhdwNDK4CLWtG7kZIrBsa5R.png?width=1200)

「 **Vibe Codingでゼロからサイトを作ると、実際どうなるのか？** 」という実験を兼ねて、ポートフォリオを作り直しました。  
これを始めたときはV0にかなりハマっていて、全体の9割ほどをV0で構築しています。

DB連携、管理画面、問い合わせ機能まで一通り実装してみて、学びは多かった一方で、プラットフォームのありがたみも強く実感しました…💭

![画像](https://assets.st-note.com/img/1766239843-7H9sOwtWRnzMmhSDP4folFya.png?width=1200)

管理用のAdmin画面

> 💻️ 使ったツール: V0 / Cursor / Figma  
> 📚️ テーマ: Vibe Codingでのデザイン精度検証と、CMS的ダッシュボード・外部連携  
> 🔗 URL: [https://sakinotomiura.com/](https://sakinotomiura.com/)

### 🗓️ 月間/年間振り返りツール

![画像](https://assets.st-note.com/img/1766277978-gc0x8vaNFXMYju9We2EOpJQR.png?width=1200)

（中身は一時的にダミーテキスト入れてます）

以前下記のnoteにもまとめた、月間振り返りのシステムを、「見た目をもう少し自分好みにしたい」という衝動から、ゼロから作り直しました。

要件定義はこれまでの運用をベースにしつつ、 **情報設計以外はFigmaを使わず、言語化だけでどこまでイメージに近づけられるか** を試しています。

スクリーンショットをアップロードすると自動で数値が反映されるなど、Notion時代よりも、より自分専用のツールになりました。

> 💻️ 使ったツール: Cursor / ChatGPT  
> 📚️ テーマ: 複雑な要件を、Figmaなしでどこまで具体化できるか

### 🌳 ブロック3D盆栽

![画像](https://assets.st-note.com/production/uploads/images/237834143/picture_pc_68067000294c4c51b71c601d08a261cc.gif?width=1200)

完全に遊びのプロジェクトですが、3Dブロックで構成された盆栽を育てられるアプリです。

盆栽の種類や鉢のデザインを変更でき、GitHubのアカウントを連携すると **GitHubのコントリビューションに応じて盆栽が成長したり、葉が少なくなったり** します🌳

> 💻️ 使ったツール: V0 / Cursor  
> 📚️ テーマ: 3Dボクセル生成・変形と、GitHub Contributionsの可視化

  

### 🍷 ワインメニューアプリ

![画像](https://assets.st-note.com/production/uploads/images/237720554/picture_pc_914f9b139ad3a819fdd30cfacb82a73f.gif?width=1200)

社内イベントでワイン選定を任されるようになったことをきっかけに、ワインラインナップを一覧で見られるメニューアプリを作りました🍷

「その場で撮影した写真」や「オンラインショップで購入したワインの画像」など、 **ソースが違う写真を同じイラストフォーマットで出力するプロンプト** がなかなかの学びポイントでした。Nano Bananaで生成するのに json でかなり細かく指定を入れています。

> 💻️ 使ったツール: Cursor / Nano Banana Pro  
> 📚️ テーマ: 写真から一貫したイラスト表現を生成するためのプロンプト設計と、操作感の気持ちよさ

### 🎛️ UI Lab

![画像](https://assets.st-note.com/production/uploads/images/237841279/picture_pc_1a230305a29c992cde5bf7950fb3ef5f.gif?width=1200)

![画像](https://assets.st-note.com/img/1766240601-Q4PlKgCkLXhmpoiO1TI967yM.png?width=1200)

同じく会社関連のイベントで、 [UI Lab](https://studio.connpass.com/event/373723/) というコミュニティのロゴ/VIを担当する機会がありました。VIやモーション自体はAI使わず、ひたすらコンセプト分解やイテレーションを重ねています。

このツールでは、第一回のテーマ「ノブ」に合わせて、ロゴで遊べるツールをV0で制作しました🎛️

> 💻️ 使ったツール: V0 / Figma  
> 📚️ テーマ：イベントテーマを起点に、ロゴを「操作できる体験」へ落とし込むための構造設計と実装実験  
> 🔗 URL: [https://ui-lab-logo-experiments.vercel.app/](https://ui-lab-logo-experiments.vercel.app/)

### 🧪 Text to SVGアイコン研究

![画像](https://assets.st-note.com/img/1766282673-Kh0gxL1QByuArz5Pp6Wj72fG.png?width=1200)

振り返れば1年以上前から定期的にやっている、「 **プロンプトからSVGアイコンを生成する** 」リサーチをするためのツールです。

Sonnet / Opus / GPT 5.2…などいろいろなモデルで、例えば「ペン」などのシンプルなプロンプトから生成されるアイコンの違いを比べるためのツールです。

> 💻️ 使ったツール: Cursor  
> 📚️ テーマ: 実務視点で見た、AIアイコン生成の成立条件

  

## デザインと実装の距離感が変わった半年

この半年を通して、「デザイナーであること」が一番楽しい状態に戻れた気がします。  
実際に動くものを自分の手で作れることが、エンジニアとしての専門的な経験がない自分にとって、デザインの楽しさを一段と広げてくれました。

Figmaのデータ、紙とペンのスケッチ、言葉にして考えること…それらと同じように、コードも自然と同じデザインプロセスの中に置かれ始めた感覚があります。

Vibe Codingが、自分にとってデザインと思考を前に進めるための、ひとつの手段になった半年でした。  
  
2026年もたくさんデザインしていきたいなと思います。
